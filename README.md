# 📂 Domain Models

## Student

Represents a student in the system.

Properties include:

* Name
* Email
* Age
* Grade
* BirthDate

Relationships:

* Belongs to one Department
* Has one Grade record
* Has many Attendance records
* Can borrow many Books

---

## Department

Represents an academic department.

Relationship:

* One Department → Many Students

---

## Grade

Stores subject grades for a student.

Subjects:

* Physics
* Chemistry
* Biology

Relationship:

* One Student ↔ One Grade

---

## Attendance

Stores student attendance information.

Implemented to practice:

* Table Mapping
* Schema Mapping
* Column Mapping
* Ignored Properties

Examples:

* Custom table name
* Custom schema
* Custom column name
* NotMapped / Ignore

---

## Book

Represents books available for students.

Relationship:

* Many-to-Many with Students

---

## StudentBook

Junction table used to implement the Many-to-Many relationship between Students and Books.

---

## Invoice

Entity created to practice advanced EF Core configurations.

Implemented concepts:

* Computed Columns
* Default Values
* SQL Expressions
* Database Generated Values

Examples:

* CustomerFullname generated automatically
* Total generated from Quantity × Price
* CreatedDate generated by SQL Server
* Default Quantity value

---

# 🔗 Relationships Implemented

## One-to-One

Student ↔ Grade

```text
Student (1) -------- (1) Grade
```

## One-to-Many

Department ↔ Students

```text
Department (1) -------- (*) Students
```

## One-to-Many

Student ↔ Attendances

```text
Student (1) -------- (*) Attendances
```

## Many-to-Many

Students ↔ Books

```text
Student (*) -------- (*) Book
```

Implemented using a junction entity:

```text
StudentBook
```
